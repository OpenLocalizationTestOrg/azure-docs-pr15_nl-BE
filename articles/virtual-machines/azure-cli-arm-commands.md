# <a name="azure-cli-commands-in-resource-manager-mode"></a>Azure CLI-opdrachten in de modus voor bronbeheer

In dit artikel wordt de syntaxis en opties voor Azure opdrachtregelinterface (CLI) opdrachten die u meestal gebruikt maken en beheren van Azure bronnen in het implementatiemodel Azure Resource Manager. U toegang tot deze opdrachten met behulp van de CLI in Resource Manager (arm) modus. Dit is geen volledige informatie over en de CLI versie kan enigszins verschillende opdrachten of parameters weergeven. Zie [Azure Resource Manager-overzicht](../azure-resource-manager/resource-group-overview.md)voor een algemeen overzicht van Azure resources en resourcegroepen.  

Ophalen is gestart, eerst [de Azure CLI installeren](../xplat-cli-install.md) en [verbinding maken met uw Azure abonnement](../xplat-cli-connect.md) met behulp van een werk of school-account of een Microsoft-account-id.

Huidige opdrachtsyntaxis en -opties op de opdrachtregel in de Resource Manager modus, typt u `azure help` kunt u help weergeven voor een specifieke opdracht `azure help [command]`. Ook vinden in de documentatie van CLI-voorbeelden voor het maken en beheren van specifieke Azure services.

Optionele parameters worden tussen vierkante haken weergegeven (bijvoorbeeld, `[parameter]`). Alle andere parameters zijn vereist.

Naast de optionele parameters opdracht specifieke hier beschreven, zijn er drie optionele parameters die kunnen worden gebruikt om gedetailleerde uitvoer als aanvraag opties en statuscodes weer te geven. De `-v` parameter biedt uitgebreide uitvoer en de `-vv` parameter geeft zelfs meer uitgebreide uitvoer. De `--json` optie wordt het resultaat in de indeling Onbewerkte json.

## <a name="setting-the-resource-manager-mode"></a>De bronnenbeheerder modus instellen

Gebruik de volgende opdracht opdrachten in de modus Azure CLI bronbeheer inschakelen.

    azure config mode arm

>[AZURE.NOTE] Van de CLI Azure Resource Manager en Azure Service Management modus zijn die elkaar wederzijds uitsluiten. Dat wil zeggen kunnen niet bronnen die zijn gemaakt in een modus worden beheerd uit de andere modus.

## <a name="azure-hdinsight-commands-to-manage-your-hdinsight-clusters"></a>Azure hdinsight: opdrachten voor het beheren van clusters HDInsight

**Opdrachten voor het maken of toevoegen aan een cluster-configuratiebestand**

    hdinsight config create [options] <configFilePath> <overwrite>
    hdinsight config add-config-values [options] <configFilePath>
    hdinsight config add-script-action [options] <configFilePath>

Voorbeeld: Maak een configuratiebestand met de scriptactie van een worden uitgevoerd wanneer u een cluster maakt.

    hdinsight config create "C:\myFiles\configFile.config"
    hdinsight config add-script-action --configFilePath "C:\myFiles\configFile.config" --nodeType HeadNode --uri <scriptActionURI> --name myScriptAction --parameters "-param value"

**Opdracht voor het maken van een cluster in een resourcegroep**

    hdinsight cluster create [options] <clusterName>
