Dit artikel bevat een voorbeeld configuratie voor het configureren van Azure VM-uitbreidingen voor Windows VMs.

## <a name="sample-template-snippet-for-vm-extensions-with-iaas-vms"></a>Voorbeeld sjabloon fragment voor VM-extensies met IaaS VMs.
Het fragment van de sjabloon voor het implementeren van extensies ziet er als volgt:

      {
      "type": "Microsoft.Compute/virtualMachines/extensions",
      "name": "MyExtension",
      "apiVersion": "2015-05-01-preview",
      "location": "[parameters('location')]",
      "dependsOn": ["[concat('Microsoft.Compute/virtualMachines/',parameters('vmName'))]"],
      "properties":
      {
      "publisher": "Publisher Namespace",
      "type": "extension Name",
      "typeHandlerVersion": "extension version",
      "autoUpgradeMinorVersion":true,
      "settings": {
      // Extension specific configuration goes in here.
      }
      }
      }

### <a name="customscript-extension-14"></a>CustomScript uitbreiding 1.4.
      {
          "publisher": "Microsoft.Compute",
          "type": "CustomScriptExtension",
          "typeHandlerVersion": "1.4",
          "settings": {
              "fileUris": [
                  "http: //Yourstorageaccount.blob.core.windows.net/customscriptfiles/start.ps1"
              ],
              "commandToExecute": "powershell.exe -ExecutionPolicy Unrestricted -start.ps1"
          },
          "protectedSettings": {
            "storageAccountName": "yourStorageAccountName",
            "storageAccountKey": "yourStorageAccountKey"
          }
      }

#### <a name="parameter-description"></a>Beschrijving van de parameter:

- fileUris: door komma's gescheiden lijst met URL's van de bestanden die door de uitbreiding van de VM worden gedownload. Er zijn geen bestanden worden gedownload als u niets opgeeft. Als de bestanden in Azure, opslag zijn, de fileURLs kan worden gemarkeerd als privé en de bijbehorende storageAccountName en storageAccountKey kunnen worden doorgegeven als persoonlijke parameters voor toegang tot deze bestanden.
- commandToExecute: [verplichte Parameter]: dit is de opdracht die wordt uitgevoerd door de extensie.
- storageAccountName: [optionele Parameter]: accountnaam Storage voor toegang tot de fileURLs, als deze zijn gemarkeerd als privé.
- storageAccountKey: [optionele Parameter]: sleutel van opslag Account voor toegang tot de fileURLs, als deze zijn gemarkeerd als privé.

### <a name="customscript-extension-17"></a>CustomScript uitbreiding 1.7.

Raadpleeg de CustomScript versie 1.4 voor de beschrijving van de parameter. Versie 1.7 introduceert ondersteuning voor het verzenden van parameters(commandToExecute) script als protectedSettings, in welk geval ze worden gecodeerd alvorens te verzenden. de parameter 'commandToExecute' kan worden opgegeven in instellingen of protectedSettings maar niet in beide.

        {
            "publisher": "Microsoft.Compute",
            "type": "CustomScriptExtension",
            "typeHandlerVersion": "1.7",
            "settings": {
                "fileUris": [
                    "http: //Yourstorageaccount.blob.core.windows.net/customscriptfiles/start.ps1"
                ],
                "commandToExecute": "powershell.exe -ExecutionPolicy Unrestricted -start.ps1"
            },
            "protectedSettings": {
              "commandToExecute": "powershell.exe -ExecutionPolicy Unrestricted -start.ps1",
              "storageAccountName": "yourStorageAccountName",
              "storageAccountKey": "yourStorageAccountKey"
            }
        }

### <a name="dsc-extension"></a>DSC-extensie.
      {
          "publisher": "Microsoft.Powershell",
          "type": "DSC",
          "typeHandlerVersion": "2.1(Recommendation is to use the latest version)",
          "settings": {
              "ModulesUrl": "https://UrlToZipContainingConfigurationScript.ps1.zip",
              "SasToken": "Optional : SAS Token if ModulesUrl points to Azure Blob Storage",
              "ConfigurationFunction": "ConfigurationScript.ps1\\ConfigurationFunction",
              "Properties": {
                  "ParameterToConfigurationFunction1": "Value1",
                  "ParameterToConfigurationFunction2": "Value2",
                  "ParameterOfTypePSCredential1": {
                      "UserName": "UsernameValue1",
                      "Password": "PrivateSettingsRef:Key1(Value is a reference to a member of the Items object in the protected settings)"
                  },
                  "ParameterOfTypePSCredential2": {
                      "UserName": "UsernameValue2",
                      "Password": "PrivateSettingsRef:Key2"
                  }
              }
          },
          "protectedSettings": {
              "Items": {
                  "Key1": "PasswordValue1",
                  "Key2": "PasswordValue2"
              },
              "DataBlobUri": "optional : https: //UrlToConfigurationData.psd1"
          }
      }
